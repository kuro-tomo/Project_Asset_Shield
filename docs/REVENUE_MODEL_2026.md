# 収益構造と評価額モデル (2026版)

## 1. 概要
本ドキュメントは、QuantConnect（Alpha Market/Streams）およびQuantiacsのプラットフォームを活用した収益モデルと企業価値評価の基本モデルを定義する。

## 規約・制約の参照
事業運用上の必須規約・制約は [`MASTER_CONSTITUTION.md`](../MASTER_CONSTITUTION.md) を一次情報とする。

## 2. フェーズ戦略 (Phasing)

いきなり300億円規模の運用（Phase 1）を目指すのではなく、個人規模で実行可能な実績作り（Phase 0）から開始する。

### Phase 0: Bootstrap (実績証明フェーズ)
*   **目的:** 監査可能なトラックレコード（運用実績）の構築。プラットフォーム上での公開実績を形成する。
*   **期間:** 2026 Q1 - Q4 (12ヶ月)
*   **手段:**
    1.  **Asset Shield (Current):**
        *   対象: 日本株 (J-Quants)
        *   資金: 自己資金 (Proprietary Trading)
        *   証明: フォワードテストの成績データ（改ざん不能なログ）
    2.  **QuantConnect / Alpha Market/Streams:**
        *   対象: 米国株/Crypto
        *   目的: グローバルな公開ランキングでの実績構築（※要コード移植）
*   **目標:** Sharpe Ratio > 1.5, Max Drawdown < 10% の達成。

### Phase 1: Expansion (プラットフォーム収益拡大)
*   **前提条件:** Phase 0での実績証明完了。

## 3. 前提条件 (Assumptions - Phase 1)
* **収益源:** Alpha Market購読料 + Quantiacsパフォーマンス・フィー
* **目標リターン:** 年利 +10%（保守的設定）
* **評価指標:** 安定性、低回転率、実現可能性、ドローダウン管理

## 4. 収益シミュレーション (Phase 1)

### 4.1 収益源の内訳

| 収益源 | 内容 | 備考 |
| :--- | :--- | :--- |
| **Alpha Market購読料** | 戦略の購読による月額/年額収益 | 公開ランキング上位で増加 |
| **Quantiacs運用報酬** | パフォーマンス・フィー | Allocation獲得が前提 |

## 5. コスト構造と営業利益 (EBITDA)

### 5.1 想定コスト (OpEx)
プラットフォーム収益化では、固定費を抑え、実績構築と運用維持に集中する。

* **インフラ/データ費用:** 2,000万円 (AWS, J-Quants等)
* **人件費/外注費:** 5,000万円 (少数精鋭チーム)
* **監査・コンプライアンス:** 1,500万円 (任意対応)

### 5.2 利益目標
* **年間EBITDA:** 実績拡大と購読増加に比例して上積み

## 6. 企業価値評価 (Valuation)

この収益モデルに基づく評価額試算。

* **評価指標:** 購読数の増加率、Allocation規模、公開実績の安定性
* **想定評価額:** 実績成長率とプラットフォーム内評価に連動

## 7. リスク要因と対策

1.  **市場流動性 (Liquidity):**
    *   300億円の運用はマーケットインパクトを伴う。アルゴリズムは `Slippage Aware` である必要がある。
2.  **ドローダウン:**
    *   成功報酬はHigh-Water Mark（最高益更新）が条件となるため、一度大きく負けると翌年の成功報酬が消滅するリスクがある。
    *   対策：ボラティリティコントロールによるダウンサイド保護。
3.  **コンプライアンスコスト:**
    *   プラットフォーム審査の補強資料（監査・証明）に想定外コストが発生する可能性がある。
