# 50万円からのAsset Shield起動計画 (Bootstrap Execution Plan)

## 1. 概要: 「運用」ではなく「事業開発」へ

手元資金50万円で、Asset Shieldのような機関投資家グレードのシステムを運用し、生活費を稼ぐことは不可能です。日本株の単元制度（最低購入額）と、分散投資の原則に反するからです。

したがって、この50万円は「株式運用」に使うのではなく、**「Asset Shieldという事業価値を高め、プラットフォームでの実績構築を行うための開発・検証費用」**として投下するのが唯一の合理的解決策です。

**目標:** 6ヶ月以内に、監査可能なトラックレコード（運用実績）を作り上げ、外部資金を導入できる状態にする。

## 1.1 規約・制約の参照
本計画における運用規約・制約（リスク管理、監査、セキュリティなど）は [`MASTER_CONSTITUTION.md`](../MASTER_CONSTITUTION.md) を一次情報とします。

## 2. 技術的解決策: Bootstrap Modeの実装

少額資金・初期段階での実績作りを加速させるため、システムに以下の改修を行いました。

### 2.1 スクリーニング基準の緩和 ([`src/shield/screener.py`](../src/shield/screener.py))
従来の「Institutional Mode」は、時価総額数千億円以上の超優良企業のみを対象としており、エントリーチャンスが稀でした。
新たに実装した **「Bootstrap Mode」** では、以下のように基準を緩和し、中小型成長株もターゲットに含めることで、短期間での実績作りを可能にしました。

| 指標 | 旧設定 (Institutional) | **新設定 (Bootstrap)** | 意図 |
| :--- | :--- | :--- | :--- |
| **Z-Score** | 3.0以上 (倒産確率ほぼ0) | **1.81以上** (グレーゾーン可) | 倒産リスクを許容し、リターンを追求 |
| **F-Score** | 8点以上 (満点に近い) | **6点以上** (改善傾向) | 完璧な財務より、変化率を重視 |
| **PEG** | 1.0以下 | **2.0以下** | 成長株の許容 |

### 2.2 設定の変更方法
[`src/shield/strategy_config.py`](../src/shield/strategy_config.py) にて、モードを切り替え可能です。

```python
# src/shield/strategy_config.py
SCREENING_MODE = "BOOTSTRAP"  # ここを変更
```

## 3. 50万円の最適配分 (Budget Allocation)

6ヶ月のランウェイを想定した予算配分です。

### 3.1 固定費 (計: 20万円)
*   **J-Quants API (Standard Plan):** 
    *   月額 ¥16,500 × 6ヶ月 ≈ **¥100,000**
    *   必須。これがないとシステムの優位性が証明できません。
*   **AWS インフラ:**
    *   EC2 (t3.small Spot Instance) + EBS: 月額 約¥5,000 × 6ヶ月 ≈ **¥30,000**
    *   24時間稼働のフォワードテスト環境用。スポットインスタンスを活用してコストを削減します。
*   **開発環境・ツール:**
    *   GitHub Copilot, Cursor等: 月額 ¥3,000 × 6ヶ月 ≈ **¥20,000**
    *   開発効率を最大化するため。
*   **予備費:** **¥50,000**

### 3.2 変動費・学習投資 (計: 30万円)
*   **バックテスト用計算リソース:** 
    *   大量の並列計算が必要な場合、AWS Batch等を利用。**¥50,000**
*   **技術書籍・学習:**
    *   ファイナンス理論、機械学習、Python関連。**¥50,000**
*   **残存キャッシュ (Emergency Fund):** **¥200,000**
    *   何かあった時のためのバッファ。決して相場に投入してはいけません。

## 4. 実行ロードマップ

### Phase 0.1: ローカル検証 (Cost: ¥0, 期間: 1週間)
*   **場所:** ユーザーのローカルPC (Mac)
*   **Action:** 
    *   `BOOTSTRAP` モードでのバックテストを実行。
    *   過去データで「年利20%以上」が出るパラメータセットを発見する。
    *   **ここで勝てなければ、AWSにお金を払う意味はありません。**

### Phase 0.2: フォワードテスト (Cost: ¥5,000/月, 期間: 3ヶ月)
*   **場所:** AWS
*   **Action:**
    *   システムをAWSにデプロイし、ペーパートレード（仮想売買）モードで稼働。
    *   毎日、売買シグナルと損益を改ざん不能なログとして記録。
    *   これが後の「公的証明パッケージ」の核になります。

### Phase 1: 資本配分獲得 / 収益化
*   **Action:**
    *   3ヶ月分のフォワードテスト結果と、バックテスト結果をセットにした「公的証明パッケージ」を作成。
    *   QuantConnect/Quantiacs での公開実績と合わせて提示。

50万円は「種銭」ではなく「武器を作るための予算」です。
